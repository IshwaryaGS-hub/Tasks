Multiplexing

HTTP1.1

Uses a single connection for each request, which can lead to a phenomenon known as head-of-line blocking. If one resource is slow to load, it can block the loading of other resources.

HTTP2

Supports multiplexing, allowing multiple requests and responses to be sent and received in parallel over a single connection. This helps to overcome the head-of-line blocking issue and improves overall page loading speed.

Binary Protocol

HTTP1.1

Text-based protocol, which is human-readable but can be less efficient in terms of parsing and processing.

HTTP2

Binary protocol, which is more efficient for machines to parse and results in faster processing.

Resource Prioritization

HTTP1.1

No built-in support for prioritizing resources. All resources are treated equally.

HTTP2

Supports resource prioritization, allowing the server to specify the importance of different resources. This helps in loading critical resources first, improving page rendering speed.

Objects

  • In JavaScript, objects are a fundamental data type that allows you to store and organize data using key-value pairs.
  • Objects can represent complex entities and are central to the language's flexibility and expressiveness.
  • When it comes to their internal representations, understanding how JavaScript handles objects is crucial for efficient programming.

Internal Representations

Properties

Objects in JavaScript consist of properties, where each property has a key and a corresponding value. The key is always a string or a symbol, and the value can be of any data type, including other objects.

Prototypes

JavaScript uses a prototype-based inheritance model. Objects can inherit properties and methods from other objects through a prototype chain. If a property or method is not found on an object, JavaScript looks for it in the object's prototype, and so on, until the end of the chain.
